# FlipEdu 콘텐츠 관리 시스템 — PRD (Product Requirements Document)

## 1. 제품 개요

FlipEdu 에디터와 연동되는 교육 콘텐츠 관리 시스템으로, 교사/관리자가 **쉐도잉 학습 자료**와 **학습지(문제지)**를 생성·관리할 수 있는 웹 애플리케이션입니다.

- **대상 사용자**: 학원 교사, 관리자
- **플랫폼**: 웹(PC + 모바일 반응형)
- **언어**: 한국어 UI
- **테마**: 라이트 모드 기본

---

## 2. 인증 (공통)

### 2.1 FlipEdu 다단계 로그인

| 단계 | 설명 | API |
|------|------|-----|
| 1단계 | 학원명 검색 → 브랜드 선택 (로고 표시) | `GET /api/auth/partners?name=학원명` |
| 2단계 | 지점(branch) 목록 표시 및 선택 | `GET /api/auth/branches?brandNo=X` |
| 3단계 | 아이디 + 비밀번호 입력 후 로그인 | `POST /api/auth/login` |

- 인증 토큰은 서버 세션에 저장 (`express-session`)
- 미인증 시 모든 페이지에서 로그인 화면으로 리다이렉트
- 관리자 계정(`{만능}` 접두사)은 왕관 배지 표시
- 세션 유지 확인: `GET /api/auth/me`
- 로그아웃: `POST /api/auth/logout`

## 3. 나만의 쉐도잉 (Shadowing)

### 3.1 개요
학생이 원어민 발음을 듣고 따라 읽는 쉐도잉 학습 자료를 생성·관리하는 기능입니다.

### 3.2 카테고리 관리

| 기능 | 설명 |
|------|------|
| 트리 구조 | 최대 4단계 깊이의 계층형 카테고리 |
| 생성 | 루트 또는 하위 카테고리 추가 |
| 이름 변경 | 카테고리명 수정 |
| 삭제 | 카테고리 삭제 |
| 검색 | 카테고리명으로 필터링 |
| 펼침/접기 | 트리 노드 토글 |

- API: `GET/POST/PUT/DELETE /api/flip-categories`
- FlipEdu 외부 API와 동기화 (LMS → Editor 이중 폴백)

### 3.3 쉐도잉 자료 목록 (홈 화면)

| 기능 | 설명 |
|------|------|
| 페이지네이션 | 페이지 단위로 자료 목록 표시 |
| 카테고리 필터 | 사이드바에서 카테고리 선택 시 해당 자료만 표시 |
| 검색 | 자료명으로 검색 |
| 테이블 표시 | 제목, 문장 수, 작성자, 생성일 컬럼 |
| 모바일 대응 | 카테고리 사이드바 → 오버레이 드로어로 전환 |

### 3.4 쉐도잉 자료 상세/수정

| 기능 | 설명 |
|------|------|
| 상세 보기 | 클릭 시 슬라이드아웃 패널에 문장 목록 표시 (영어 원문 + 한국어 번역) |
| 인라인 편집 | 편집 모드 토글 → 자료 제목 및 개별 문장 직접 수정 |
| 단건/일괄 삭제 | 선택한 자료 삭제 |
| 카테고리 이동 | 선택한 자료를 다른 카테고리로 이동 |

### 3.5 쉐도잉 자료 생성

| 기능 | 설명 |
|------|------|
| 제목 입력 | 자료 제목 설정 |
| 카테고리 선택 | 트리 드롭다운에서 카테고리 지정 |
| 텍스트 붙여넣기 & 분리 | 긴 텍스트를 문장 단위 또는 문단 단위로 자동 분리 |
| 수동 입력 | 개별 문장 추가/삭제/순서 변경(위/아래) |
| 번역 입력 | 각 문장의 원문(`originalText`) + 번역(`translation`) 입력 |
| 질문 설정 | 개별 문장의 질문 프롬프트 수정 (기본값: "아래 문장을 읽고 녹음하세요.") |
| 질문 일괄 변경 | 전체 문장의 질문을 한번에 변경 |
| AI 음성/채점 | `aiSound`, `aiGrading` 데이터 자동 생성 |

- API: `POST /api/shadowing/create` → 개별 항목 생성 후 `shadowing-paper`로 그룹핑
- 외부 API 이중 폴백: LMS(`lms.flipedu.net`) 우선 → Editor(`editor.flipedu.app`) 보조

## 4. 나만의 학습지 (Worksheet)

### 4.1 개요
객관식/주관식 문제로 구성된 학습지를 생성·관리하는 기능입니다.

### 4.2 카테고리 관리

| 기능 | 설명 |
|------|------|
| 트리 구조 | 최대 4단계 깊이의 계층형 카테고리 |
| 생성 | 루트 또는 하위 카테고리 추가 |
| 이름 변경 | 카테고리명 수정 |
| 삭제 | 카테고리 삭제 |
| 검색 | 카테고리명으로 필터링 |

- API: `GET/POST/PUT/DELETE /api/question-paper-categories`

### 4.3 학습지 목록 (홈 화면)

| 기능 | 설명 |
|------|------|
| 페이지네이션 | 페이지 단위로 학습지 목록 표시 |
| 카테고리 필터 | 사이드바에서 카테고리 선택 시 필터링 |
| 검색 | 학습지명으로 검색 |
| 삭제 | 학습지 삭제 (확인 다이얼로그 포함) |
| 스켈레톤 로딩 | 데이터 로딩 중 스켈레톤 UI 표시 |

### 4.4 학습지 생성 (모달)

전체 화면 모달 (1100px 너비) — 좌측: 문항 편집, 우측: 도구 패널

#### 4.4.1 기본 설정

| 항목 | 설명 |
|------|------|
| 학습지 제목 | 필수 입력 |
| 카테고리 | 트리 드롭다운에서 선택 |

#### 4.4.2 문항 유형

| 유형 | 설명 |
|------|------|
| 객관식 (CHOICE) | 질문 + 보기(최대 5개) + 정답 번호 선택 |
| 주관식 (SHORT_ANSWER) | 질문 + 정답 텍스트 입력 |

#### 4.4.3 문항별 입력 필드

| 필드 | 설명 | 필수 |
|------|------|------|
| 질문(question) | 문제 내용 | O |
| 지문(body) | 긴 지문/본문 | X |
| 보기(choices) | 객관식 선택지 (최대 5개) | 객관식만 |
| 정답(answer/correctIndex) | 정답 | O |
| 해설(explanation) | 풀이 설명 | X |
| 태그(tags) | 분류 태그 | X |
| 문항별 카테고리 | 개별 문항에 카테고리 지정 가능 | X |

#### 4.4.4 주관식 채점 옵션

| 옵션 | 설명 |
|------|------|
| 대/소문자 구분 | `gradingCaseSensitive` |
| 특수기호 포함 | `gradingSpecialChars` |
| 띄어쓰기 구분 | `gradingSpacing` |
| OR 채점 | `gradingOr` — 여러 정답 허용 |

#### 4.4.5 문항 조작

| 기능 | 설명 |
|------|------|
| 추가 | 새 빈 문항 추가 |
| 복제 | 기존 문항 복사하여 추가 |
| 삭제 | 개별 문항 삭제 |
| 순서 변경 | 위/아래 이동 |

#### 4.4.6 우측 도구 패널 (340px)

**텍스트로 일괄 추가**
- 객관식: 탭 구분 — `질문\t보기1\t보기2\t보기3`
- 주관식: 탭 구분 — `질문\t정답\t해설`
- 여러 줄 붙여넣기로 한번에 다수 문항 생성

**일괄 설정**
- 전체 객관식으로 변경
- 전체 주관식으로 변경
- 전체 문항에 현재 카테고리 적용
- 문항 순서 섞기
- 전체 초기화

**현황**
- 총 문항 수
- 객관식/주관식 비율 실시간 표시

#### 4.4.7 미리보기

- 모달 하단 "미리보기" 버튼 클릭 시 오버레이 팝업
- 완성된 학습지 형태로 문항 미리보기
- 문항 유형 배지, 카테고리 배지, 보기/정답/해설 표시

### 4.5 API 플로우

```
학습지 생성 요청
  ├─ 1) 각 문항을 개별 question으로 등록 (POST /api/branch/questions)
  │    └─ questionType, question, choices, answer, classifyNo 등
  └─ 2) 생성된 question들을 묶어 paper 등록 (POST /api/branch/question-paper)
       └─ title, classifyNo, questionNos[]
```

- 외부 API 이중 폴백: LMS 우선 → Editor 보조
- `subjectGroup`: 항상 `"eng"`

## 5. 공통 기술 사양

### 5.1 프론트엔드

| 항목 | 기술 |
|------|------|
| 프레임워크 | React + TypeScript |
| 번들러 | Vite (HMR) |
| 라우팅 | Wouter |
| 상태관리 | TanStack React Query v5 |
| UI 컴포넌트 | Shadcn/ui + Radix UI |
| 스타일링 | Tailwind CSS |
| 애니메이션 | Framer Motion |
| 폼 | React Hook Form + Zod |
| 아이콘 | Lucide React |
| 폰트 | Inter (본문), Calistoga (디스플레이) |

### 5.2 백엔드

| 항목 | 기술 |
|------|------|
| 프레임워크 | Express 5 (Node.js) |
| 런타임 | tsx (dev), esbuild (prod) |
| DB | PostgreSQL + Drizzle ORM |
| 검증 | Zod (프론트/백엔드 공유 스키마) |
| 세션 | express-session + connect-pg-simple |
| 외부 API | FlipEdu LMS / Editor API 프록시 |

### 5.3 반응형 디자인

| 환경 | 동작 |
|------|------|
| PC | 좌측 카테고리 사이드바 + 우측 컨텐츠 영역 |
| 모바일 (< 768px) | 사이드바 → Sheet 드로어, 테이블 컬럼 축소, 도구 패널 숨김 |

### 5.4 외부 API 연동 패턴

```
요청 → LMS (lms.flipedu.net) 시도
  ├─ 성공 → 응답 반환
  └─ 실패 → Editor (editor.flipedu.app) 폴백 시도
       ├─ 성공 → 응답 반환
       └─ 실패 → 에러 반환
```

- 모든 외부 API 호출 시 서버가 프록시 역할 (CORS 회피)
- 인증 쿠키/토큰 서버 세션에서 자동 첨부
